AI / NPC
Pathfinding Bot

Uses PathfindingService (A* internally) to build a navmesh from all walkable parts at runtime. ComputeAsync() returns waypoints — the bot walks them with Humanoid:MoveTo(), skipping timed-out waypoints instead of restarting from scratch.

PathfindingService A* Navmesh ComputeAsync Humanoid API
AI / Movement
Smart Movement

Path is computed once and walked start to finish. At each waypoint the angle between directions is measured — over 35° counts as a turn, slowing to WalkSpeed 7 with a 40% chance to pause and look around. A straightCount tracks consecutive straight waypoints — after 3 in a row it sprints at WalkSpeed 20.

Angle Detection Dynamic WalkSpeed Straight Count Behavioural Logic
Security
Server-Side Anti-Cheat

Runs every 0.1s server-side across all players. Speed check flags over 32 studs/sec. Teleport check flags jumps over 50 studs in one tick. Flying check raycasts 4 studs down — sustained airtime over 4s triggers a flag. At 3 violations: log, warn, or kick. Whitelisted usernames are never kicked.

Server-Side Raycasting Violation Stack Whitelist Heartbeat Loop
Backend
DataStore — Coin System

Each player's coins saved under player_USERID. On join GetAsync loads data and fires it to the client HUD via RemoteEvent. On leave SetAsync writes it back. Auto-saves every 60s. BindToClose catches server shutdowns. All calls wrapped in pcall — a failed save never crashes the game. Client never touches the DataStore directly.

DataStoreService pcall Safety BindToClose RemoteEvents Auto-Save
Game Loop + AI
1v1 Arena & Combat AI

CharacterAutoLoads disabled — the round manager owns all spawning. Reads ArenaZone bounds dynamically. Five-state combat AI: CHASE → CIRCLE → ATTACK → DODGE → LUNGE, ticking every 0.03s. Pathfinding runs in its own task.spawn thread so reactions are never blocked. Lunge fires between 8–14 studs on a 1.8s cooldown. All movement clamped to arena bounds.
